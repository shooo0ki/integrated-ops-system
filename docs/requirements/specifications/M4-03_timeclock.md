# M4-03 打刻画面

## 目的
出勤・退勤の打刻と日報の入力を完了する。S1 シナリオの核心画面。1 分以内に完了できる UX が必須。

## 対象ペルソナ / シナリオ
- 山田（インターン）・鈴木（社員）: S1 毎日の打刻 + 日報 Slack 通知

## UI 構成

### 出勤前（未打刻）
- 現在時刻の大きな表示
- 「出勤する」ボタン（大きく目立つ）
- 「今日やること」テキストエリア（必須）
- Slack 通知先チャンネル表示

### 出勤後・退勤前
- 出勤時刻表示
- 「退勤する」ボタン
- 休憩時間入力（分）
- 「今日やったこと」テキストエリア（必須）
- 「明日やること」テキストエリア（必須）

### 退勤後
- 本日の勤務サマリー（出勤・退勤・実働時間）
- 「勤怠一覧を見る」リンク

## 入力項目
| 項目 | 必須 | 制約 | 例 |
|------|------|------|----|
| 今日やること（出勤時） | 必須 | 最大 500 文字 | `〇〇機能の実装` |
| 休憩時間（退勤時） | 必須 | 0 以上の整数（分） | `60` |
| 今日やったこと（退勤時） | 必須 | 最大 500 文字 | `〇〇機能を完成させた` |
| 明日やること（退勤時） | 必須 | 最大 500 文字 | `テストを書く` |

## 操作
| 操作 | 結果 |
|------|------|
| 出勤ボタン押下 | 現在時刻で出勤記録 + Slack 通知 |
| 退勤ボタン押下 | 現在時刻で退勤記録 + 実働時間算出 + Slack 通知 |

## Slack 通知内容
### 出勤時
```
[氏名] が出勤しました（HH:MM）
本日のタスク: [今日やること]
```

### 退勤時
```
[氏名] が退勤しました（HH:MM）
勤務時間: X時間Y分
実績: [今日やったこと]
明日の予定: [明日やること]
```

## エラー / 例外
| 条件 | 挙動 |
|------|------|
| 「今日やること」未入力で出勤ボタン | 「今日やることを入力してください」 |
| Slack 通知失敗 | 打刻は成功。「Slack 通知に失敗しましたが、打刻は記録されました」トースト |
| 二重出勤（すでに出勤済） | 「本日はすでに出勤済みです」 |
| 退勤時の必須項目未入力 | 各項目に「入力してください」バリデーション |

## 権限（ロール別）
- 全ロール: 自分の打刻のみ操作可

## 関連データ（エンティティ）
- `ATTENDANCE`、`MEMBER`

## 関連 API
- `GET /api/attendance/today`
- `POST /api/attendance/clock-in`
- `POST /api/attendance/clock-out`
