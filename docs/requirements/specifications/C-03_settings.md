# C-03 設定画面

## 目的
システム全体のマスタデータ・Slack 連携設定・ロール管理などを行う管理者専用画面。

## 対象ペルソナ / シナリオ
- 佐藤（管理者）: 初期設定・運用変更

## UI 構成
- タブ構成
  - Slack 連携設定
  - 会社情報・掛け率
  - ロール・権限設定（将来拡張）
  - システム情報（バージョン）

## 入力項目
| 項目 | 必須 | 制約 | 例 |
|------|------|------|----|
| Slack Webhook URL | 必須 | URL 形式 | `https://hooks.slack.com/...` |
| 勤怠通知チャンネル | 必須 | Slack チャンネル名 | `#attendance` |
| 月末締め通知日 | 必須 | 1〜28 の整数 | `25` |
| 会社名（親会社） | 必須 | 最大 50 文字 | `ブーストコンサルティング` |
| 会社名（子会社） | 任意 | 最大 50 文字 | `SALT2` |
| **掛け率（markup）** | **必須** | **1.00 以上の小数（小数点第 2 位まで）** | `1.20` |

> **掛け率メモ**: Boost 案件における「SALT2 の Boost への請求額 = 人件費 × 掛け率 + ツール費」の乗数。
> デフォルト値は 1.20（人件費の 20% 上乗せ）。PJ 別 PL 画面（M6-01）のシミュレーション機能と連動。

## 表示項目
- 現在の設定値一覧
- Slack 連携テスト結果（成功 / 失敗）

## 操作
| 操作 | 説明 |
|------|------|
| 保存 | 設定値を DB に保存 |
| Slack 接続テスト | Webhook にテストメッセージを送信し疎通確認 |

## エラー / 例外
| 条件 | 挙動 |
|------|------|
| Webhook URL 不正 | 「有効な Slack Webhook URL を入力してください」 |
| Slack 送信失敗 | 「Slack への接続に失敗しました。URL を確認してください」 |

## 権限（ロール別）
| ロール | アクセス |
|--------|---------|
| 管理者 | フルアクセス |
| その他 | アクセス不可（404 または権限エラー） |

## 関連データ（エンティティ）
- `SYSTEM_CONFIG`（設定テーブル: キー・バリュー形式）
  - キー例: `slack_webhook_url`, `slack_channel`, `closing_day`, `company_parent`, `company_child`, `markup_rate`

## 関連 API
- `GET /api/settings`
- `PUT /api/settings`
- `POST /api/settings/slack/test`
