# M6-01 PJ 別 PL

## 目的
プロジェクト単位の月次損益（売上・コスト・粗利）を確認し、手動コストを入力する。S3 のアサイン後に人件費影響を確認する。

## 対象ペルソナ / シナリオ
- 佐藤（管理者）: 月次 PL 確認・売上入力
- 田中（マネージャー）: 担当 PJ の PL 確認

## UI 構成
- PJ 選択ドロップダウン
- 対象月選択
- PL テーブル（売上・コスト・粗利）
- 月次推移グラフ
- 売上・手動コスト入力フォーム

## 表示項目（PL テーブル）
| 区分 | 項目 | 入力方式 |
|------|------|---------|
| 売上 | 月額契約売上 | 手動入力 |
| 売上 | 追加売上 | 手動入力 |
| コスト | 人件費（月額メンバー） | 自動計算（アサイン工数 × 月額按分） |
| コスト | 人件費（時給メンバー） | 自動計算（勤怠実績 × 時給） |
| コスト | 外注費 | 手動入力 |
| コスト | その他経費 | 手動入力 |
| 合計 | 粗利 | 自動計算（売上 − コスト合計） |
| 合計 | 粗利率 | 自動計算 |

## 入力項目
| 項目 | 必須 | 制約 | 例 |
|------|------|------|----|
| 月額契約売上 | 任意 | 0 以上の整数 | `500000` |
| 追加売上 | 任意 | 0 以上の整数 | `100000` |
| 外注費 | 任意 | 0 以上の整数 | `80000` |
| その他経費 | 任意 | 0 以上の整数 | `10000` |

## 操作
| 操作 | 説明 |
|------|------|
| 月切り替え | 前月・翌月ナビ |
| 売上・コスト入力 | 該当セルを直接編集 |
| 保存 | 手動入力値を保存 |

## エラー / 例外
| 条件 | 挙動 |
|------|------|
| 勤怠未集計の月の人件費 | 「勤怠データが未集計です。M5-01 で集計後に確認してください」 |

## 権限（ロール別）
| ロール | 閲覧 | 入力 |
|--------|------|------|
| 管理者 | 全 PJ | ○ |
| マネージャー | 担当 PJ のみ | 担当 PJ のみ |
| 社員・インターン | 不可 | 不可 |

## 関連データ（エンティティ）
- `PL_RECORD`、`PROJECT`、`PROJECT_ASSIGNMENT`、`ATTENDANCE`、`MEMBER`

## 関連 API
- `GET /api/pl/project/:id?month=`
- `PUT /api/pl/project/:id`
