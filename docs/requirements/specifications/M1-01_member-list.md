# M1-01 メンバー一覧

## 目的
全メンバーの一覧を表示し、検索・フィルタリングで目的のメンバーを素早く見つける。

## 対象ペルソナ / シナリオ
- 佐藤（管理者）: メンバー管理・確認
- 田中（マネージャー）: アサイン候補の確認
- 鈴木（社員）: 同僚の基本情報確認

## UI 構成
- 検索バー（氏名・メールアドレス）
- フィルター（ステータス / ロール）
- メンバーカード or テーブル一覧（**所属会社ラベルは表示しない**）
- 新規追加ボタン（管理者のみ表示）

> **設計メモ**: メンバー画面では Boost / SALT2 ラベルは表示しない。
> 会社ラベルはプロジェクト画面のみ表示する。

## 入力項目（検索・フィルター）
| 項目 | 必須 | 制約 | 例 |
|------|------|------|----|
| 氏名 / メール（フリーワード） | 任意 | 部分一致 | `田中` |
| ステータス | 任意 | 役員 / 社員 / 本採用インターン / 研修インターン / 研修メンバー | — |
| ロール | 任意 | 管理者 / マネージャー / メンバー | — |

> **ロール統合メモ**: 社員とインターンは権限が同じため「メンバー」に統合する。
> 表示上の役職・雇用形態はプロフィール項目として保持するが、権限制御は同一として扱う。

## 表示項目（一覧行）
| 項目 | 全ロール | 管理者のみ |
|------|---------|-----------|
| 氏名 | ○ | — |
| プロフィール画像 | ○ | — |
| ステータス（役職・雇用形態） | ○ | — |
| メールアドレス | — | ○ |
| 入社日 | — | ○ |

## 操作
| 操作 | 説明 | 遷移先 |
|------|------|--------|
| 行クリック | メンバー詳細を開く | M1-02 |
| 新規追加ボタン | 登録フォームを開く | M1-03 |
| フィルター適用 | 一覧を即時絞り込み | — |

## エラー / 例外
| 条件 | 挙動 |
|------|------|
| 検索結果 0 件 | 「該当するメンバーが見つかりません」 |
| データ取得失敗 | エラートースト + リトライボタン |

## 権限（ロール別）
| ロール | 閲覧範囲 |
|--------|---------|
| 管理者 | 全員・全項目 |
| マネージャー | 全員（基本情報のみ） |
| メンバー（社員・インターン共通） | 全員（基本情報のみ） |

## 関連データ（エンティティ）
- `MEMBER`、`USER_ACCOUNT`

## 関連 API
- `GET /api/members?status=&role=&q=`
