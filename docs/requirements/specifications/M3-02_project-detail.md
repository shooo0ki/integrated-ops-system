# M3-02 プロジェクト詳細

## 目的
プロジェクトの詳細情報・ポジション定義・アサイン状況を一覧で確認する。S3 シナリオの起点。

## 対象ペルソナ / シナリオ
- 佐藤（管理者）・田中（マネージャー）: S3 PJ ポジション空き確認

## UI 構成
- プロジェクト情報ヘッダー（名称・ステータス・期間・クライアント）
- ポジション一覧テーブル（必要人数・現在のアサイン・空き状況）
- アサイン済みメンバーリスト（ポジション・工数・期間）
- 編集ボタン（管理者・マネージャー）
- アサイン管理ボタン

## 表示項目
| 項目 | 説明 |
|------|------|
| PJ 名・説明・ステータス | — |
| 所属会社・クライアント名 | — |
| 開始日・終了予定日 | — |
| 契約形態・月額契約金額 | 管理者のみ |
| ポジション名・必要人数・充足人数 | — |
| アサインメンバー（名前・ポジション・月間工数・期間） | — |

## 操作
| 操作 | 遷移先 |
|------|--------|
| 編集ボタン | M3-03 PJ 登録 / 編集 |
| アサイン管理ボタン | M3-04 アサイン管理 |
| メンバー名クリック | M1-02 メンバー詳細 |
| 工数管理ボタン | M3-05 工数管理 |

## エラー / 例外
| 条件 | 挙動 |
|------|------|
| 存在しない PJ ID | 404 ページ |

## 権限（ロール別）
| ロール | 閲覧 | 編集 |
|--------|------|------|
| 管理者 | フル | フル |
| マネージャー | 担当 PJ のみ | 担当 PJ のみ |
| 社員・インターン | アサイン済み PJ のみ（限定項目） | 不可 |

## 関連データ（エンティティ）
- `PROJECT`、`PROJECT_POSITION`、`PROJECT_ASSIGNMENT`、`MEMBER`

## 関連 API
- `GET /api/projects/:id`
- `GET /api/projects/:id/assignments`
