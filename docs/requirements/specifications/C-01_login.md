# C-01 ログイン画面

## 目的
ユーザーを認証してシステムへのアクセスを許可する。

## 対象ペルソナ / シナリオ
- 全ペルソナ（毎日利用）
- S1: 打刻の起点となる入口

## UI 構成
- ロゴ・システム名
- 「Slack でログイン」ボタン
- 「Google でログイン」ボタン
- フッター（バージョン情報）

## 入力項目
| 項目 | 必須 | 制約 | 例 |
|------|------|------|----|
| OAuth プロバイダー選択 | 必須 | Slack または Google | — |

## 表示項目
- システム名・ロゴ
- エラーメッセージ（認証失敗時）

## 操作
| 操作 | 説明 |
|------|------|
| Slack でログイン | Slack OAuth 2.0 フローを開始 |
| Google でログイン | Google OAuth 2.0 フローを開始 |

## エラー / 例外
| 条件 | 表示メッセージ |
|------|---------------|
| 未登録メールで認証 | 「このアカウントはシステムに登録されていません。管理者にお問い合わせください。」 |
| OAuth 認証失敗 | 「ログインに失敗しました。再度お試しください。」 |
| セッション期限切れ | ログイン画面にリダイレクト + 「セッションが切れました。再ログインしてください。」 |

## 権限（ロール別）
- 全ロール共通（認証前画面のためロール制御なし）

## 関連データ（エンティティ）
- `USER_ACCOUNT`（メールアドレス照合、ロール取得）
- `MEMBER`（アカウントに紐づくメンバー情報）

## 関連 API
- `POST /api/auth/callback/slack`
- `POST /api/auth/callback/google`
- `GET /api/auth/session`
