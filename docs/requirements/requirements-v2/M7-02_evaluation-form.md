# M7-02 人事評価入力・編集

## 目的
特定メンバーの対象月に対して PAS（Professional / Appearance / Skill）評価を入力・更新する。
1メンバー × 1ヶ月 = 1レコード（upsert）として管理する。

## 対象ペルソナ / シナリオ
- 佐藤（管理者）: 月次評価会議の結果をシステムに入力する

## UI 構成
- モーダル or 専用ページ
- メンバー名・対象月ヘッダー
- P / A / S 各軸のスコア選択（1〜5 ラジオボタン or ボタン群）
- 各軸のラベル説明（ホバーでツールチップ）
- 評価コメント（テキストエリア、任意）
- [保存] [キャンセル]

## 入力項目

| 項目 | 必須 | 制約 | 例 |
|------|------|------|----|
| 対象メンバー | 必須 | メンバー選択（新規時）/ ヘッダーに固定（編集時） | 鈴木一郎 |
| 対象月 | 必須 | YYYY-MM 形式、未来月は選択不可 | `2026-02` |
| P スコア | 必須 | 1〜5 の整数 | `4` |
| A スコア | 必須 | 1〜5 の整数 | `3` |
| S スコア | 必須 | 1〜5 の整数 | `4` |
| 評価コメント | 任意 | 最大 1000 文字 | `今月は期限内に全タスク完了。コミュニケーションも改善` |

## 表示項目
| 項目 | 説明 |
|------|------|
| 前回評価 | 直前月の P/A/S スコア（比較参考表示） |
| 変化インジケータ | ↑↓ で前月比を表示 |
| スコア説明 | 各スコア値にラベル（要改善/普通以下/標準/優秀/卓越）を表示 |

## 評価軸詳細

| 軸 | 評価観点（例） |
|----|--------------|
| P（プロフェッショナル） | 締め切り厳守、報連相、チームへの貢献、主体的な問題解決 |
| A（アピアランス） | 清潔感、服装・身だしなみ、立ち居振る舞い、対外的な印象 |
| S（スキル） | 担当業務の遂行能力、技術習熟度、成長速度、アウトプット品質 |

## 操作
| 操作 | 説明 |
|------|------|
| スコア選択 | 1〜5 をクリックで選択 |
| 保存 | upsert（同月評価済みなら上書き） |
| キャンセル | モーダルを閉じる |

## エラー / 例外
| 条件 | 挙動 |
|------|------|
| P/A/S いずれか未入力で保存 | 「スコアをすべて入力してください」バリデーション |
| 未来月を選択 | 「未来の月は選択できません」 |
| 対象メンバー未選択 | 「メンバーを選択してください」 |
| サーバーエラー | トースト「保存に失敗しました」 |

## 権限（ロール別）
| ロール | 操作 |
|--------|------|
| 管理者 | 全メンバーに対し入力・編集可 |
| マネージャー | アクセス不可（403） |
| 社員・インターン | アクセス不可（403） |

## 関連データ（エンティティ）
- `PERSONNEL_EVALUATION`
- `MEMBER`
- `USER_ACCOUNT`（evaluator_id = ログインユーザーの UserAccount.id）

## 関連 API
- `GET /api/evaluations/:memberId?month=YYYY-MM` — 前回値取得
- `POST /api/evaluations` — 新規作成
- `PUT /api/evaluations/:id` — 更新
