**1. プロジェクト概要**

**1.1 背景と目的**

ブーストコンサルティング及び子会社SALT2において、現在Googleスプレッドシートを中心に管理している人事・勤怠・プロジェクト・財務情報を統合的に管理するWebアプリケーションを構築する。本システムにより、手作業による管理コストの削減、データの一元化、業務プロセスの自動化を実現する。

**1.2 対象組織**

  ------------------ ---------------------------------------------------
  **項目**           **内容**

  親会社             ブーストコンサルティング

  子会社             SALT2

  管理対象人数       30〜100名程度

  メンバー種別       役員 / 社員 / 本採用インターン / 研修インターン /
                     研修メンバー
  ------------------ ---------------------------------------------------

**2.4 契約書管理（新規追加）**

- 採用アプリで「採用」となったメンバーを本システムへ取り込み、契約書 PDF を生成・送受信する。
- 契約書テンプレート（雛形）に対し、氏名・住所・契約日・報酬条件などを差し込み生成する。
- DocuSign API（送信/受信）を用い、アプリ画面から送付・署名リクエスト・署名完了の受信を行う。
- 締結済み契約書はメンバーごとのプロフィール画面に格納し、本人と管理者のみ閲覧できる。
- 口座番号などの個人情報も同様に本人・管理者のみ表示とする（権限制御必須）。

**2.3 権限管理**

ユーザーの権限は4段階のロールで管理し、各ロールで閲覧・操作できる機能を制御する。

  -------------- ------------------------------ ----------------------------------------------------------------------

  管理者         役員・経営層                   全機能のフルアクセス。メンバー追加・削除、システム設定変更、全PL閲覧

  マネージャー   PM・リーダー                   担当PJ管理、メンバースキル閲覧、勤怠承認、PJ単位PL閲覧

  社員           正社員                         自身の情報編集、勤怠打刻、担当PJ閲覧、全体スケジュール閲覧

  インターン     各種インターン・研修メンバー   自身の情報編集、勤怠打刻、担当PJ閲覧
  -------------- ------------------------------ ----------------------------------------------------------------------

**2.4 契約書管理（採用連携）**

- 社内の採用アプリで「採用」となったメンバーを本システムへ取り込む。
- 契約書テンプレート（雇用契約書の雛形）に氏名・住所・契約日・報酬条件を差し込み PDF を生成する。
- DocuSign API で送信・署名依頼を行い、Webhook で署名完了を受信する。
- 締結済み契約書 PDF をメンバープロフィールに紐づけて保存し、本人と管理者のみ閲覧可能にする（リンク/署名URLともに権限制御）。
- 契約ステータスは draft / sent / waiting_sign / completed / voided を管理する。
- 口座番号などの個人情報も同じく本人・管理者のみ表示。DB・API でロールチェックを必須とする。

**3. スキル管理モジュール (M2)**

**3.1 スキル体系**

スキルは「カテゴリ \>
スキル項目」の2階層で管理する。カテゴリ及びスキル項目は管理者が自由に追加・編集・並べ替え可能とする。

**3.1.1 初期スキルカテゴリとスキル項目**

  ------------------ ------------------------ ----------------------------------------------------
  **カテゴリ**       **スキル項目**           **説明**

  エンジニアリング   フロントエンド           React, Vue.js等のフロントエンド開発スキル

  エンジニアリング   バックエンド             API設計、サーバーサイド開発スキル

  エンジニアリング   インフラ                 AWS, GCP等のクラウドインフラ構築・運用スキル

  エンジニアリング   データサイエンス         データ分析、機械学習モデル構築スキル

  エンジニアリング   LLM                      大規模言語モデルの活用・ファインチューニングスキル

  ビジネス           PM力                     プロジェクトマネジメント、進捗管理、リスク管理

  ビジネス           営業力                   顧客折衝、提案活動、リレーション構築

  ビジネス           スライドライティング力   プレゼンテーション資料の構成・デザイン力

  ビジネス           提案力                   課題分析から解決策の提案まで

  テクニカル         技術理解                 技術的な概念・アーキテクチャの理解力

  テクニカル         技術提案                 技術的な解決策を顧客に分かりやすく提案する力
  ------------------ ------------------------ ----------------------------------------------------

**3.2 スキルレベル定義**

各スキル項目に対して、5段階のレベルで評価を行う。

  ------------ -------------- ------------------------------------------------------------
  **レベル**   **ラベル**     **定義**

  1            初学者         基本的な知識を学習中。指導の下で簡単なタスクが実行できる

  2            初級           基本的なタスクを自力で遂行できる。標準的な問題を解決できる

  3            中級           複雑なタスクを自力で遂行できる。他者への指導が可能

  4            上級           高度な問題解決ができる。設計・アーキテクチャの判断が可能

  5            エキスパート   該当分野の第一人者。組織全体の技術方針を策定できる
  ------------ -------------- ------------------------------------------------------------

**3.3 スキル評価フロー**

スキル評価は以下のフローで実施される。

- 管理者またはマネージャーがメンバーのスキルレベルを設定・更新する

- 各スキル評価には評価日と任意のメモ（評価根拠）を付記できる

- スキル評価の履歴は保持され、成長の推移を確認できる

- スキルマトリクス画面で全メンバーのスキルレベルを一覧表示できる

**3.4 スキルマトリクス画面**

全メンバーのスキルレベルを一覧で確認できるマトリクス画面を提供する。行にメンバー名、列にスキル項目を配置し、各セルにレベル（1〜5）を色分けして表示する。カテゴリごとのフィルタリング、特定スキルレベル以上のメンバー抽出機能を備える。

**4.4.1 スキルフィルタリング機能**

アサイン画面では以下の操作が可能。

- 対象ポジションの必要スキルが画面上部に表示される

- フィルターON: 必要スキルを満たすメンバーのみ表示

**4.5 工数管理**

メンバーごとのプロジェクト別工数を管理する。月次を基本単位とし、週次・日次での修正も可能とする。勤怠データとの紐付けにより、実際の勤務時間がどのプロジェクトに配分されているかを把握できる。

- プロジェクトアサイン時に月次の想定工数（時間 or %）を記録する（例: 40h/月, 50%）。
- 個人単位で「どのプロジェクトに何%/何時間かけているか」を月次で管理し、会社別集計（Boost / SALT2）を算出できるようにする。

**6. 請求書生成モジュール (M5)**

**6.1 月末締め処理フロー**

毎月末日に以下の処理を自動的に実行する。

- 当月の全勤怠データを集計する

- 時給制メンバーごとに「総勤務時間 x 時給」で給与を算出する

- Slackで各メンバーにメンションし、勤怠サマリーと給与見込額を通知する

- メンバーが「OK」ボタンを押すと、請求書の自動生成が実行される

- 修正が必要な場合は、管理者が勤怠データを修正し再集計する

**6.2 請求書テンプレート**

Excelテンプレートを用いて請求書を自動生成する。テンプレートには以下の項目が自動挿入される。

  ---------------- ------------------ -------------------------------------
  **項目**         **データソース**   **説明**

  請求者氏名       メンバーDB         メンバー管理の氏名

  請求者住所       メンバーDB         メンバー管理の住所

  振込先銀行名     メンバーDB         銀行名・支店名・口座番号・名義

  請求対象期間     勤怠データ         当月の開始日〜末日

  勤務明細         勤怠データ         日付ごとの勤務時間一覧

  合計勤務時間     勤怠データ         当月の総勤務時間

  時給単価         メンバーDB         設定された時給額

  請求金額         計算値             合計勤務時間 x 時給単価

  会社別請求額     計算値             個人の会社別稼働配分に基づき「Boost向け」「SALT2向け」に按分した金額

  消費税           計算値             請求金額 x 税率

  請求合計         計算値             請求金額 + 消費税

  発行日           システム           請求書生成日

  請求書番号       システム           自動採番（年月-連番）
  ---------------- ------------------ -------------------------------------

**6.3 請求書送付**

生成された請求書はSlackの本人DMに自動送付される。メンバーはこの請求書をダウンロードし、経理部門に提出する。経理部門での処理（支払い実行）はシステムの対象外とし、請求書作成までを自動化範囲とする。

**7. PL管理モジュール (M6)**

**7.1 概要**

プロジェクト単位の損益（Profit and
Loss）を管理する。売上は手動入力を基本とし、将来的に請求書ベースの自動計上に拡張する。コスト（人件費）はアサイン情報と勤怠データから自動計算される。

- PL/CFは管理者のみ閲覧可能とする。
- CF（キャッシュフロー）は当面、手入力での管理とし、将来の外部連携を検討する。

**7.2 PL構成要素**

  ---------- ------------------------ -------------- --------------------------------------------
  **区分**   **項目**                 **入力方式**   **説明**

  売上       月額契約売上             手動入力       クライアントからの月額契約金額

  売上       追加売上                 手動入力       スポット案件等の追加売上

  コスト     人件費（月額メンバー）   自動計算       アサインされた月額メンバーの稼働割合 x
                                                     月額報酬

  コスト     人件費（時給メンバー）   自動計算       勤怠データの実稼働時間 x 時給

  コスト     外注費                   手動入力       外部委託にかかる費用

  コスト     ツール費用               自動/手動       メンバーの有料ツール（月額）を会社/プロジェクト配分で計上

  コスト     社内精算（親子会社間）   自動計算       支払元会社と実稼働配分が異なる場合の内部振替（例: 所属SALT2支払、Boost稼働30%なら30%をBoostへ請求）

**7.5 社内精算ロジック（親会社・子会社）**
- 支払元会社: 給与を振り込む会社（所属会社をデフォルトとする）。
- 実稼働配分: 勤怠のプロジェクト配分合算または会社別稼働配分を参照。
- 社内精算額 = 支給総額 × 実稼働配分(支払元以外)。
- Boost⇔SALT2 間の内部請求書（社内用）を月次締め時に自動生成し、PLでは内部取引フラグ付きで計上（連結では相殺する）。
- SALT2 の案件でも Boost/SALT2 双方のPJが存在するため、会社別（Boost / SALT2）および会社ラベル別（PJの所属会社）でPLを個別表示し、合計も提示する。

  コスト     その他経費               手動入力       ツール費用、交通費等のプロジェクト関連経費
  ---------- ------------------------ -------------- --------------------------------------------

**7.3 人件費の自動計算ロジック**

**7.3.1 時給制メンバー**

時給制メンバーの人件費 = 当月の実勤務時間（勤怠データ） x
時給単価。勤怠データが複数PJにまたがる場合は、工数管理モジュールの配分比率に基づいて按分する。

**7.3.2 月額固定メンバー**

月額固定メンバーの人件費 = 月額報酬 x
当該PJへの稼働割合。稼働割合はアサイン管理で設定された月間想定工数に基づいて計算する（当該PJ工数
/ 全PJ合計工数）。

**7.4 PL画面仕様**

- プロジェクト別の月次PL表を表示

- 売上・各コスト項目・粗利・粗利率を一覧表示

- 月次推移をグラフで可視化

- 会社別（ブーストコンサルティング / SALT2）でのフィルタリング

- 複数PJを合算した会社全体のPLサマリー表示

**7.5 キャッシュフロー管理**

月次ベースでキャッシュイン（入金）とキャッシュアウト（支出）を管理する。

  ------------------ ------------------ -------------------------------------
  **区分**           **項目**           **説明**

  キャッシュイン     クライアント入金   契約に基づく月次入金

  キャッシュイン     その他入金         補助金、立替金精算等

  キャッシュアウト   給与支払い         メンバーへの月次給与支払い

  キャッシュアウト   外注費支払い       外部委託先への支払い

  キャッシュアウト   固定費             オフィス、ツール、サーバー費用等

  キャッシュアウト   その他支出         その他の経費支払い
  ------------------ ------------------ -------------------------------------

**10.3 セキュリティ要件**

- 全通信をHTTPS化する

- 認証はOAuth 2.0 / OpenID Connectを採用する

- パスワード保存はbcryptによるハッシュ化を行う

- APIは認証トークン（JWT）で保護する

- 銀行口座情報等の機密データは暗号化して保存する

- 操作ログ（監査ログ）を記録し、不正アクセスを検知可能にする

- OWASP Top 10に対応したセキュリティ対策を実施する

**10.4 運用・保守要件**

- 日次で自動バックアップを取得する

- 障害発生時の復旧目標時間（RTO）は4時間以内とする

- システムの稼働率目標は99.5%以上とする

- ログ監視・アラート通知の仕組みを導入する
